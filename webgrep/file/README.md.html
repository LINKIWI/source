      1 # webgrep
      2 
      3 webgrep is a web client for livegrep.
      4 
      5 ## Building
      6 
      7 To build a standalone `webgrep` binary,
      8 
      9 ```
     10 $ npm run build
     11 $ npm run release
     12 $ ./dist/bin/webgrep --help
     13 ```
     14 
     15 ## Configuration
     16 
     17 See `config.example.yaml` for an example configuration file. The config file location can be specified at runtime with the `--config` flag.
     18 
     19 The following documents each field and its expected value:
     20 
     21 |Key|Required|Description|
     22 |-|-|-|
     23 |`server.listen.transport`|Yes|Network transport on which to bind the server HTTP listener; one of `tcp` or `unix`|
     24 |`server.listen.address`|Yes|IP address and port pair for TCP transports, or a socket path for Unix domain socket transports|
     25 |`server.livegrep.load_balancing_policy`|No|Load balancing policy to use among multiple livegrep replicas; one of `round-robin` (rotate through all servers on each request), `failover` (prefer earlier servers, failing over to next servers on request errors) (default exclusive use of first server with no load balancing)|
     26 |`server.livegrep.request_timeout`|No|Per-request RPC invocation timeout in milliseconds (default 3000)|
     27 |`server.livegrep.servers[].authority`|No|Livegrep codesearch gRPC server HTTP/2 authority header|
     28 |`server.livegrep.servers[].address`|Yes|Livegrep codesearch gRPC server address|
     29 |`server.metrics.statsd.addr`|No|Address of the statsd server or listener for metrics reporting (default disabled)|
     30 |`server.metrics.statsd.prefix`|No|String prefix for all emitted statsd metrics (default `webgrep`)|
     31 |`server.metrics.statsd.sample_rate`|No|statsd reporting sample rate (default `1.0`)|
     32 |`server.cache.redis.ttl`|No|TTL for Redis cache keys in milliseconds (default 10 minutes)|
     33 |`server.cache.redis.timeout`|No|Timeout for all Redis requests (default disabled)|
     34 |`server.cache.redis.prefix`|No|Prefix to apply to Redis cache keys (default `webgrep`)|
     35 |`server.cache.redis.socket`|No|Redis Unix socket path|
     36 |`server.cache.memory.ttl`|No|TTL for in-memory cache keys in milliseconds (default 10 minutes)|
     37 |`server.source.local.repositories[].name`|No|Local on-disk repository name for source code viewer|
     38 |`server.source.local.repositories[].path`|No|Local on-disk repository directory path|
     39 |`server.source.gitlab.base_url`|No|GitLab instance base URL for source code viewer (default `https://gitlab.com`)|
     40 |`server.source.gitlab.socket_path`|No|Path to a Unix domain socket for accessing GitLab API|
     41 |`server.source.gitlab.access_token`|No|GitLab API access token|
     42 |`server.source.gitlab.tls_key`|No|TLS client key path for mutual authentication with GitLab|
     43 |`server.source.gitlab.tls_cert`|No|TLS client certificate path for mutual authentication with GitLab|
     44 |`server.source.gitlab.tls_ca_cert`|No|TLS CA certificate path for server authentication of GitLab|
     45 |`client.site.logo`|No|URL to an image to use as the logo shown in the upper-left corner|
     46 |`client.site.title`|No|Formatting string for creating browser page titles (default `webgrep`)|
     47 |`client.site.about`|No|About text shown in the Admin section|
     48 |`client.site.banner.title`|No|Header informational banner title|
     49 |`client.site.banner.description`|No|Header informational banner text description|
     50 |`client.site.banner.link`|No|Header informational banner external link title|
     51 |`client.site.banner.href`|No|Header informational banner external link target URL|
     52 |`client.options.metadata_poll_interval`|No|Poll interval for new server metadata in milliseconds|
     53 |`client.resources[].title`|No|Title for an arbitrary external link shown in the Admin section|
     54 |`client.resources[].href`|No|URL for an associated external link shown in the Admin section|
